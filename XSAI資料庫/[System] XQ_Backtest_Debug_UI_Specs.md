# xscript 知識庫：回測運算與系統排錯 (v2.0)

本文件詳述 xscript 回測報告的統計指標計算公式，以及 XS_UI 事件檢視器中的錯誤代碼與排除方式。資料來源為 `回測報告計算2023.docx` 與 `XS_UI 事件檢視器規格.xlsx`。

## 1. 回測報告計算公式 (Backtest Calculation Metrics)

本章節定義回測報告中各項統計數據的數學定義與計算邏輯。

### 1.1 報酬率與獲利指標

* **總報酬率 (Total Return)**
    * **公式**：$\prod (1 + R_{daily}) - 1$
    * **說明**：採複利計算。每日報酬率 $R_{daily} = \frac{\text{今日權益} - \text{昨日權益}}{\text{昨日權益}}$。

* **年化報酬率 (CAGR)**
    * **公式**：$((1 + \text{總報酬率}) ^ {1 / \text{年數}}) - 1$
    * **說明**：將總報酬率標準化為年複利成長率。若回測期間不足一年，通常直接顯示總報酬率或特別標註。

* **勝率 (Win Rate)**
    * **公式**：$\frac{\text{獲利交易次數}}{\text{總交易次數}} \times 100\%$
    * **定義**：
        * **獲利交易**：單筆交易報酬率 > 0。
        * **總交易次數**：包含所有已平倉的交易。

* **獲利因子 (Profit Factor)**
    * **公式**：$\frac{\text{總獲利金額}}{\text{總虧損金額}}$
    * **說明**：衡量策略獲利能力相對於虧損風險的指標。值大於 1 代表策略總體是獲利的。

### 1.2 風險指標

* **最大區間虧損 (Maximum Drawdown, MDD)**
    * **公式**：$\max(\frac{P - L}{P})$
    * **定義**：權益曲線從波段高點 ($P$) 回落至波段低點 ($L$) 的最大跌幅百分比。
    * **意義**：代表策略在最壞情況下可能遭受的本金損失。

* **夏普比率 (Sharpe Ratio)**
    * **公式**：$\frac{\bar{R}_p - R_f}{\sigma_p} \times \sqrt{252}$
    * **參數設定**：
        * $R_f$ (無風險利率)：回測報告中通常預設為 0 或參考當時定存利率（需確認系統設定）。
        * $\sqrt{252}$：假設一年交易日為 252 天進行年化調整。

## 2. 智慧單安控阻擋代碼 (SOOrderBlocked Error Codes)

本章節列出 `SOOrderBlocked` 事件中具體的錯誤訊息與判斷邏輯。每項錯誤代碼皆包含觸發原因與解決方案，供 AI 精準檢索使用。資料來源：`XS_UI 事件檢視器規格.xlsx - SOOrderBlocked錯誤訊息(安控).csv`。

### 錯誤代碼：CheckPrice
* **錯誤訊息**：委託價格超過漲跌停
* **觸發條件**：`OrderPrice > UpLimit` (高於漲停) 或 `OrderPrice < DownLimit` (低於跌停)
* **解決方案**：
    1.  檢查腳本中的委託價格計算邏輯是否合理。
    2.  若策略需要追價，請勾選策略屬性中的「委託價格可超出漲跌停限制」選項 (2025/09/19 新增功能)。

### 錯誤代碼：CheckAmount
* **錯誤訊息**：進場金額超過上限
* **觸發條件**：`StrategyAmount + OrderAmount > MaxAmount` (策略累計已進場金額 + 本筆委託金額 > 設定上限)
* **解決方案**：
    1.  檢查策略屬性中的「每日進場金額上限」設定值。
    2.  減少單筆下單數量，或檢查是否策略重複進場導致額度用盡。

### 錯誤代碼：CheckFrequency
* **錯誤訊息**：每分鐘交易次數過多
* **觸發條件**：`TradeCount_1min > MaxFreq` (該策略在過去 1 分鐘內的交易次數 > 設定上限)
* **解決方案**：
    1.  檢查腳本邏輯，確認是否在 0 與 1 部位間快速震盪 (High Frequency Oscillation)。
    2.  放寬策略屬性中的「每分鐘最多交易次數」設定 (2024/11/29 新增功能)。

### 錯誤代碼：CheckPositionLimit
* **錯誤訊息**：超過商品最大部位限制
* **觸發條件**：`abs(CurrentPosition + OrderQty) > MaxPosition` (預期部位絕對值 > 設定上限)
* **解決方案**：
    1.  檢查策略屬性中的「單一商品最大部位限制」。
    2.  確認腳本是否重複觸發加碼訊號 (Over-Stacking)。

### 錯誤代碼：CheckEntryCount
* **錯誤訊息**：超過每日最多進場次數
* **觸發條件**：`DailyEntryCount >= MaxEntryCount` (且本次交易被判定為新進場行為)
* **解決方案**：
    1.  檢查策略屬性中的「單一商品每日最多進場次數」。
    2.  確認策略是否過於頻繁地進行「進場->平倉->進場」的操作。

### 錯誤代碼：CheckDirection (做空限制)
* **錯誤訊息**：不允許作空
* **觸發條件**：`AccountDirection == LongOnly` (帳號或策略限多) 且 `OrderDirection == Short` (下空單)
* **解決方案**：
    1.  檢查策略屬性中的「多空方向」設定。
    2.  確認交易帳號權限是否允許融券或期貨做空。

### 錯誤代碼：CheckDirection (做多限制)
* **錯誤訊息**：不允許作多
* **觸發條件**：`AccountDirection == ShortOnly` (帳號或策略限空) 且 `OrderDirection == Long` (下多單)
* **解決方案**：
    1.  檢查策略屬性中的「多空方向」設定。
    2.  確認是否誤觸做多訊號。

### 錯誤代碼：CheckMargin
* **錯誤訊息**：保證金不足
* **觸發條件**：`AvailableMargin < RequiredMargin` (帳戶可用權益數 < 該筆委託所需原始保證金)
* **解決方案**：
    1.  帳戶權益數不足，請進行入金。
    2.  減少下單口數，或改操作保證金較低的商品。

### 錯誤代碼：CheckOrderType
* **錯誤訊息**：不支援的委託種類
* **觸發條件**：帳號類型不支援該商品或委託屬性 (例如：現股當沖帳號在非當沖時段下單)
* **解決方案**：
    1.  檢查帳號類型 (證券/期貨) 與下單商品是否匹配。
    2.  確認下單時段是否符合該商品的交易規則 (如盤中/盤後)。

## 3. 事件檢視器顯示欄位 (Event Viewer Specs)

本章節定義事件檢視器中各欄位的顯示內容與來源，協助用戶理解 Log 結構。資料來源：`XS_UI 事件檢視器規格.xlsx - 顯示內容.csv`。

### 欄位詳解
* **日期 (Date)**
    * 定義：事件發生的日期。
    * 範例：`2023/10/27`
* **時間 (Time)**
    * 定義：事件發生的 Server 時間 (精確到毫秒)。
    * 範例：`13:00:01.500`
* **策略名稱 (Strategy Name)**
    * 定義：觸發事件的策略名稱。
    * 範例：`MACD均線策略`
* **商品 (Symbol)**
    * 定義：商品名稱與代碼。
    * 範例：`台積電(2330)`
* **訊息 (Message)**
    * 定義：核心欄位，顯示 `Info`, `Warning` 或 `Error` 的具體內容。若為安控阻擋，此處會顯示如「委託價格超過漲跌停」等訊息。
* **價格 (Price)**
    * 定義：當時委託單的價格。
    * 範例：`550.0`
* **數量 (Qty)**
    * 定義：當時委託單的數量。
    * 範例：`1`
* **部位 (Position)**
    * 定義：事件發生後的策略部位狀態。
    * 範例：`1` (代表多單 1 口)
* **代碼 (Code)**
    * 定義：系統內部的錯誤代碼 (Error Code)，用於工程師除錯。
    * 範例：`CheckPrice`
