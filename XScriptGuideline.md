# XScript 腳本寫作規範

以下規範旨在提供一套一致且易於理解的 XScript 腳本撰寫標準，既方便大規模範例庫檢索與 LLM 學習，也利於開發者閱讀與維護。

---

## 零、絕對語法與撰寫原則

在進入具體的架構與邏輯之前，所有開發者必須嚴格遵守以下兩項最高原則：

- **限定使用 XS 語言**：本規範與撰寫環境嚴格限定僅能使用 XS (XScript) 語言。絕對禁止混用或使用其他任何程式語言。
- **一律英文小寫原則**：程式碼中的**所有英文字母一律強制使用小寫**。例如，必須寫成 `getfield` 而非 `GetField`，寫成 `setposition` 而非 `SetPosition`。

---

## 一、XS 核心運作原理與五大腳本精神

在撰寫腳本前，必須先理解系統的底層運作邏輯，以及各類別腳本的專屬定位。

### 底層運作原理

- **時間序列與 K 棒運算**：XS 執行時的觀念類似於看 K 線圖，系統會提供一段歷史的 K 棒序列，然後由最舊的資料（左）往最新的日期（右），一根一根地執行腳本內的邏輯。
- **逐筆洗價**：預設洗價通常是在一根 K 棒收盤時運算一次，但在盤中即時監控時，每收到一筆新的成交 tick，就會觸發一次 K 棒的洗價。
- **目標部位驅動**：XS 的自動交易並非傳統的「送出買進 1 張的委託」，而是基於「期望持倉變成多少」的部位導向設計。系統每次洗價時，會自動比對目前的預期部位 (`position`) 與實際庫存 (`filled`)，算出落差後再送出委託單。

### 五大腳本核心精神

- **指標腳本**：在 K 線圖或副圖上繪製線圖、區塊或標記，將數值運算結果「視覺化」，輔助觀察趨勢。
- **選股腳本**：從商品池中篩選符合條件的標的，精神在於「精準過濾」與「條件排序」。
- **警示腳本**：盤中即時監控，條件觸發時彈出視窗、發送聲音或手機推播，精神在於「即時狀態通知」。
- **交易腳本**：執行程式交易的回測或實單自動交易，直接影響資金進出，輸出語法最為嚴格，精神在於「資金控管」與「精確的部位調度」。
- **函數腳本**：將共用的運算邏輯打包，提供給其他四種腳本呼叫，精神在於「邏輯模組化」與「程式碼重用」。

## 二、腳本四大區塊架構與排版規範

每一支 XS 腳本在撰寫時，都必須遵循以下「四大區塊」的基本架構：

1. **文首說明區**：使用標準註解格式標明腳本名稱、邏輯說明、資料來源版權宣告。
2. **變數宣告區**：所有 `input` 與 `var`/`vars` 統一在此宣告，每行末尾用 `//` 說明意義。
3. **邏輯判斷區**：資料取值、指標運算與條件判斷。禁止超過 3 層巢狀 `if`，請拆為 `condition1`、`condition2` 等布林變數。
4. **腳本輸出區**：依據判斷區結果執行對應動作（`ret = 1`、`plot`、`setposition`），與邏輯區分開。

```xs
// ============================================================
// 腳本名稱：{依使用者需求描述總結主旨後填入}
// 邏輯說明：{依使用者需求描述總結大意後填入}
// ============================================================
// 此代碼參考資料感謝 XQ 達人園長、院長、貓老大、老墨和馬克提供
// ============================================================

// ------------------------------
// 1. 變數宣告區
// ------------------------------

// ------------------------------
// 2. 邏輯判斷區
// ------------------------------

// ------------------------------
// 3. 腳本輸出區
// ------------------------------
```

## 三、變數宣告與命名規範

- **命名防護機制**：宣告 `input` 和 `vars` 變數名稱時，前方必須加上底線符號 `_`（例如：`_period`, `_day`）。
- **一律小寫**：遵循第零節最高原則，所有變數名稱必須全部小寫（例如：`_fastperiod`, `_slowperiod`, `_mavalue`）。
- **陣列宣告的 1002 錯誤防呆**：單一陣列元素上限約 7000 個。**抓取 tick 的二維陣列，最安全的寫法為 `array: _ticks[500, 10](0);`**。
- **狀態保存 (`intrabarpersist`)**：凡涉及 tick 之間累計數值的變數，必須使用 `intrabarpersist` 宣告。
- **獨立空間宣告限制**：`rank` 區塊無法使用外部 `input`，需在內部建立專屬變數。
- **厳禁使用的名稱**：系統自動忽略的字（如 a, is, on, the）及系統內建常數（如 `pi`, `monday`）不可宣告為變數。

## 四、五大類別腳本輸出語法與限制

- **指標腳本**：使用 `plot1` 到 `plot99` 輸出數值。搭配 `setplotlabel` 動態設定名稱。進階可使用 `plotline`、`plotfill` 填色，或 `noplot` 中斷連線。
- **選股腳本**：條件成立時必須設定 `ret = 1;`。使用 `outputfield` 輸出數值。若需排序，使用獨立的 `rank` 區塊，透過 `retval` 輸出排序依據。
- **警示腳本**：條件成立時設定 `ret = 1;` 觸發警示。可使用 `alert` 產生自訂警示紀錄。**絕對限制：警示腳本無法使用 `outputfield` 函數**。
- **交易腳本**：**絕對限制：嚴禁使用 `ret = 1;` 或 `plot` 作為輸出**。必須使用 `setposition`（推薦）或 `buy`/`sell` 控制部位。必須附加 `label:="自訂字串"` 區分觸發條件。
- **函數腳本**：將最終結果賦值給「函數名稱」本身以傳回數值。若需同時輸出多個結果，在 `input` 中加上 `numericref`、`stringref` 等傳址參數。

## 五、防呆與除錯機制

- 在使用歷史資料前，檢查資料長度

```xs
if currentbar < _slowperiod then
    return;  // 資料不足，退出
```

- 對於除零等例外情況，加入預設值或警告

```xs
if volume = 0 then
    _avgvolume = 0
else
    _avgvolume = totalvolume / volume;
```

### 資料取值防呆 (`default`)

務必在 `getfield` 後方加上 `default` 參數，避免資料缺漏時造成腳本中斷：

```xs
value1 = getfield("本益比", "d", default:=0);
```

### 跨頻率取值正確寫法

若要取跨頻率的前期資料，必須直接對函數加上期數：

```xs
// 正確：直接對 getfield 加上中括號取前期值
value1 = getfield("月營收", "m")[1];

// 錯誤：先存入變數後再加中括號（會產生對位問題）
value1 = getfield("月營收", "m");
value2 = value1[1]; // 嚴禁此寫法
```

### 交易進場互斥判斷

進場時，務必同時判斷 `position = 0 and filled = 0`，代表「策略預期沒有部位，且實際也沒有部位」時才執行進場，以防重複送單。

### 停損停利價格基準

不要使用當時的 `close`，應使用 `filledavgprice`（實際成交均價）作為計算基準，避免滑價造成出場點計算錯誤。

### 運行環境防呆

為防止策略被放錯頻率執行，腳本第一行應加上頻率保護：

```xs
if barfreq <> "min" then raiseruntimeerror("僅支援分鐘頻率");
```

當沖腳本必須加入嚴格的時間範圍限制與尾盤平倉防呆機制。

### readticks 函數的絕對格式

盤中抓取逐筆明細的核心函數 `readticks` 具備嚴格限制：

- **參數限制**：僅接受 2 個參數：`readticks(陣列名稱, 紀錄序號的變數)`。禁止輸入 `date` 或 `time` 等舊版參數。
- **維度限制**：傳入陣列的第二維度**強制 >= 10**（塞入11個隱藏欄位），否則編譯失敗。
- **欄位對位**：`[i, 3]` 成交價、`[i, 4]` 單筆量、`[i, 5]` 內外盤標記 (1買/-1賣)、`[i, 10]` 連續成交總量。

### 換日歸零防呆

- **一般環境**：當使用 `readticks` 或需要在同根 K 棒內不斷累加數據時，必須在邏輯區最上方加入 `if date <> date[1] then begin ... end;` 來將所有累加器與序號變數洗白為 0，確保新的一天重新計算。

在逐筆洗價 (tick) 環境下，**絕對禁止依賴 `date[1]`** 做換日判斷。必須宣告獨立的 `intrabarpersist` 變數記錄「最後歸零日」：

```xs
var: intrabarpersist _reset_date(0);

if _reset_date <> date then begin
    // 執行各項累計數值的歸零動作...
    _reset_date = date; // 歸零後立即上鎖
end;
```

### 開盤集合競價（第一筆）處理

台灣股市 09:00:00 開盤第一筆為「集合競價」，內外盤標記通常為 0。`readticks` 回傳陣列索引 `[1]` 是「時間最新」的一筆，欲精準抓取時間序列上的「第一筆」，迴圈必須使用 `downto 1`（從最舊往最新讀取）：

```xs
var: intrabarpersist _has_first_tick(0);

if _reset_date <> date then begin
    _has_first_tick = 0;
    _reset_date = date;
end;

for _idx = _row_count downto 1 begin
    if _tick_array[_idx, 2] >= 090000 and _tick_array[_idx, 11] > 0 then begin
        if _has_first_tick = 0 then begin
            _bs_flag = 1; // 強制當作外盤
            _has_first_tick = 1;
        end else begin
            _bs_flag = _tick_array[_idx, 5]; // 後續看真實標記
        end;
    end;
end;
```

### 除錯輸出 (`print`)

開發期間建議使用 `print()` 將關鍵變數與布林值印出抓蟲。若在交易腳本中使用 `print`，必須透過 `file` 參數指定輸出檔案路徑與名稱。**效能警告**：印出大量資料會大幅拖慢雲端伺服器的回測速度，建議只在小範圍抓漏時開啟，正式跑長天期回測前務必將 `print` 註解掉。

## 六、資料範圍控制：settotalbar 與 setbackbar

這兩個函數用於控制腳本執行時的歷史 k 棒資料準備量，直接影響指標計算的正確性與跨頻率運算的效能。

### 函數寫法範例

settotalbar (設定資料讀取總筆數)

用來設定腳本執行時系統需準備的歷史 k 棒總數（不含當日即時洗價 k 棒）。

```xs

// 警示腳本預設只有 200 筆，若要計算 240 日均線必須強制加大讀取筆數

settotalbar(300);

value1 = average(close, 240);

```

setbackbar (設定最大引用筆數 / 跨頻率優化)

當腳本引用的歷史期數極深，或進行跨頻率運算時，預先告訴系統準備歷史緩衝區，能極大幅度減少系統不必要的運算。

```xs

// 在 5 分鐘頻率腳本中，計算日線的 20 日均線

input: _davglength(20, "日均線期數");

// 效能優化寫法：直接宣告需要引用 20 筆的「日」資料

setbackbar(_davglength, "d");

// 主頻率(5分k)只要極少 k 棒即可成功執行

value1 = average(getfield("close", "d"), _davglength);

```

### 開發注意事項

1. 預設值差異：若未寫入 settotalbar，系統會依腳本類型給予預設值：指標腳本為「全部歷史長度」、警示腳本為「200筆」、選股腳本為「10筆」。
1. 效能拿捏：settotalbar 設太大會明顯拖慢策略啟動與洗價速度（影響 for 迴圈總次數）；而在跨頻率抓取長天期資料時，強烈建議使用 setbackbar(期數, "頻率") 來取代放大 settotalbar，讀取速度會大幅提升。

## 七、商品清單 (group) 應用

當策略從單一商品擴展到「一籃子商品」（如計算族群總營收、抓取個股所有權證），就必須使用商品清單。

### 函數寫法範例

```xs

// 宣告清單變數

group: _symbolgroup();

var: _sum(0), _i(0);

// 自動取得台灣50的成分股

_symbolgroup = getsymbolgroup("tse50.sj", "成分股");

value1 = groupsize(_symbolgroup); // 防呆：取得成分股總檔數

// 針對清單執行迴圈運算前，務必先確認清單內有商品

if value1 > 0 then begin

for _i = 1 to value1 begin

// 檢查該商品是否有"月營收"欄位，再進行跨商品取值

if checksymbolfield(_symbolgroup[_i], "月營收", "m") then begin

_sum += getsymbolfield(_symbolgroup[_i], "月營收", "m");

end;

end;

end;

```

### 開發注意事項

1. 索引值規定：清單內容的索引值從 1 開始，不可使用 0。若在腳本中直接印出 group 變數 (未加索引)，會導致「索引值超出範圍」的錯誤。
1. 唯讀屬性：清單是唯讀的 (read-only)，只能讀取代號，嚴禁在腳本內修改（如 `_symbolgroup = "2330.tw"` 會編譯失敗）。
1. 容量上限：一個 group 清單最多只能容納 3000 個商品。
1. 執行時機：處理清單的程序發生在「洗價之前」，系統會先建立好清單才開始執行歷史 k 棒的運算。

## 八、rank 排行運算 (選股專用)

rank 是一個獨立的運算區塊，用於選股範圍內所有商品的橫向（cross-sectional）排名，能將策略從「絕對數值」昇華到「相對強弱」判斷。

### 函數寫法範例

```xs

// 1. 在最外層宣告排行物件 (預設由大到小排序)

rank _myrank begin

// 必須在區塊內獨立宣告與抓取變數

value1 = getfield("外資持股比例", "d");

value2 = value1 - value1[1]; // 修正原稿邏輯：與昨日相比的差值

retval = value2; // 指定以此數值作為排行依據

end;

// 2. 主邏輯：只挑選排名前 10 名的股票

if _myrank.pos <= 10 then ret = 1;

// 3. 輸出屬性供介面查看

outputfield(1, _myrank.value, 2, "外資持股增加比例");

outputfield(2, _myrank.pos, 0, "外資買超排名");

```

### 開發注意事項 (四大鐵律)

1. 僅限選股：此功能僅限「選股腳本」使用，指標、警示、交易皆不支援。
1. 必須放在最外層 (top level)：rank 宣告絕對不可以被包在 if ... then 判斷式或 for 迴圈裡面。
1. 獨立變數空間 (scope 隔離)：無法讀取腳本外部的 input 參數或自訂變數，所有需要的變數必須在 rank 區塊內重新抓取或運算。
1. 屬性呼叫格式：呼叫如 .pos (名次)、.value (數值)、.range (百分比) 等屬性時，小數點前後不可以有空白。名稱不可與外部變數重複。

## 九、綜合寫法範例

### 選股腳本（四大區塊標準寫法）

```xs
// ============================================================
// 腳本名稱：外資連買且突破均線選股
// 邏輯說明：篩選外資連續買超3天，且今日收盤價突破自訂均線的股票
// ============================================================
// 此代碼參考資料感謝 XQ 達人園長、院長、貓老大、老墨和馬克提供
// ============================================================

// ------------------------------
// 1. 變數宣告區
// ------------------------------
input:
    _length(10, "均線天數");   // 外部參數：均線計算天期

var:
    _mavalue(0),               // 儲存均線數值
    _foreignbuy(0);            // 儲存外資買賣超張數

// ------------------------------
// 2. 邏輯判斷區
// ------------------------------
_mavalue = average(close, _length);
_foreignbuy = getfield("外資買賣超", "d");

condition1 = close cross over _mavalue;
condition2 = trueall(_foreignbuy > 0, 3);

// ------------------------------
// 3. 腳本輸出區
// ------------------------------
if condition1 and condition2 then begin
    ret = 1;
    outputfield1(_mavalue, "均線數值");
    outputfield2(_foreignbuy, "今日外資買超張數");
end;
```

### 交易腳本（四大區塊標準寫法）

```xs
// ============================================================
// 腳本名稱：均線交叉交易策略
// 邏輯說明：依據快慢均線交叉進行交易
// ============================================================
// 此代碼參考資料感謝 XQ 達人園長、院長、貓老大、老墨和馬克提供
// ============================================================

// ------------------------------
// 1. 變數宣告區
// ------------------------------
input:
    _fastperiod(20, "快線天數"),
    _slowperiod(50, "慢線天數");

var:
    _fastma(0),                // 快線均線
    _slowma(0),                // 慢線均線
    _isbullishcross(false);    // 是否黃金交叉

// ------------------------------
// 2. 邏輯判斷區
// ------------------------------
if currentbar < _slowperiod then return;

_fastma = average(close, _fastperiod);
_slowma = average(close, _slowperiod);
_isbullishcross = _fastma crosses above _slowma;

// ------------------------------
// 3. 腳本輸出區
// ------------------------------
if _isbullishcross and position = 0 and filled = 0 then begin
    setposition(1, label:="均線黃金交叉");
end;


```

## 十、交付後使用指引

當完成 XS 程式碼撰寫並通過審查後，必須引導用戶完成以下步驟：

### 1. 匯入 XQ

提供兩種匯入方式讓用戶選擇：

- **方式 A（直接貼上）**：複製程式碼 → 打開 XQ 全球贏家 → 策略(D) → XScript 編輯器(E) → 新增腳本（類型須對應腳本類別）→ 貼上程式碼。
- **方式 B（開啟 .xs 檔）**：打開 XQ 全球贏家 → 策略(D) → XScript 編輯器(E) → 檔案 → 開啟 XS 檔案 → 選擇對應的 .xs 檔。

### 2. 編譯

告知用戶按 **F6** 或點選工具列「編譯」按鈕。編譯失敗時，請用戶將錯誤訊息貼回來協助修正。

### 3. 開始使用

根據腳本類別，告知用戶對應的啟用操作：

- **指標腳本**：點選「加入指標」，在技術分析圖上顯示
- **選股腳本**：點選「加入選股」，在選股中心使用
- **警示腳本**：點選「加入雷達」，在策略雷達中監控
- **交易腳本**：點選「加入自動交易」，設定自動下單策略

### 4. 詢問存檔

交付後必須詢問用戶：「是否要另存成 .xs 檔案？」

- 用戶選「是」→ 詢問存檔路徑，將程式碼存為 .xs 檔，並提示可用方式 B 匯入
- 用戶選「不用」→ 結束
