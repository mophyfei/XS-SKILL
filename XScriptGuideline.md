# XScript 腳本寫作規範

以下規範旨在提供一套一致且易於理解的 XScript 腳本撰寫標準，既方便大規模範例庫檢索與 LLM 學習，也利於開發者閱讀與維護。

---

## 零、XS 核心運作原理與五大腳本精神

在撰寫腳本前，必須先理解系統的底層運作邏輯，以及各類別腳本的專屬定位。

### 底層運作原理

- **時間序列與 K 棒運算**：XS 執行時的觀念類似於看 K 線圖，系統會提供一段歷史的 K 棒序列，然後由最舊的資料（左）往最新的日期（右），一根一根地執行腳本內的邏輯。
- **逐筆洗價**：預設洗價通常是在一根 K 棒收盤時運算一次，但在盤中即時監控時，每收到一筆新的成交 tick，就會觸發一次 K 棒的洗價。
- **目標部位驅動**：XS 的自動交易並非傳統的「送出買進 1 張的委託」，而是基於「期望持倉變成多少」的部位導向設計。系統每次洗價時，會自動比對目前的預期部位 (`position`) 與實際庫存 (`filled`)，算出落差後再送出委託單。

### 五大腳本核心精神

- **指標腳本**：在 K 線圖或副圖上繪製線圖、區塊或標記，將數值運算結果「視覺化」，輔助觀察趨勢。
- **選股腳本**：從商品池中篩選符合條件的標的，精神在於「精準過濾」與「條件排序」。
- **警示腳本**：盤中即時監控，條件觸發時彈出視窗、發送聲音或手機推播，精神在於「即時狀態通知」。
- **交易腳本**：執行程式交易的回測或實單自動交易，直接影響資金進出，輸出語法最為嚴格，精神在於「資金控管」與「精確的部位調度」。
- **函數腳本**：將共用的運算邏輯打包，提供給其他四種腳本呼叫，精神在於「邏輯模組化」與「程式碼重用」。

### 五大類別腳本輸出語法與限制

- **指標腳本**：使用 `plot1` 到 `plot99` 輸出數值。搭配 `setplotlabel` 動態設定名稱。進階可使用 `plotline`、`plotfill` 填色，或 `noplot` 中斷連線。
- **選股腳本**：條件成立時必須設定 `ret = 1;`。使用 `outputfield` 輸出數值。若需排序，使用獨立的 `rank` 區塊，透過 `retval` 輸出排序依據。
- **警示腳本**：條件成立時設定 `ret = 1;` 觸發警示。可使用 `alert` 產生自訂警示紀錄。**絕對限制：警示腳本無法使用 `outputfield` 函數**。
- **交易腳本**：**絕對限制：嚴禁使用 `ret = 1;` 作為輸出**。必須使用 `setposition`（推薦）或 `buy`/`sell` 控制部位。必須附加 `label:="自訂字串"` 區分觸發條件。
- **函數腳本**：將最終結果賦值給「函數名稱」本身以傳回數值。若需同時輸出多個結果，在 `input` 中加上 `numericref`、`stringref` 等傳址參數。

## 一、檔案與專案結構

- 每個策略或指標存放於單一檔案，檔名採 **PascalCase**，並以功能命名
  例如：`MovingAverageCross.eld`、`RSIOverbought.eld`
- 專案目錄下統一放置：
  - `Indicators/`：技術指標
  - `Strategies/`：交易策略
  - `Libraries/`：共用函式庫

### 腳本四大區塊架構

每一支 XS 腳本在撰寫時，都必須遵循以下「四大區塊」的基本架構：

1. **文首說明區**：使用標準註解格式標明腳本名稱、邏輯說明、資料來源版權宣告。
2. **變數宣告區**：所有 `input` 與 `var`/`vars` 統一在此宣告，每行末尾用 `//` 說明意義。
3. **邏輯判斷區**：資料取值、指標運算與條件判斷。禁止超過 3 層巢狀 `if`，請拆為 `condition1`、`condition2` 等布林變數。
4. **腳本輸出區**：依據判斷區結果執行對應動作（`ret = 1`、`plot`、`setposition`），與邏輯區分開。

```xs
// ============================================================
// 腳本名稱：{依使用者需求描述總結主旨後填入}
// 邏輯說明：{依使用者需求描述總結大意後填入}
// ============================================================
// 此代碼參考資料感謝 XQ 達人園長、院長、貓老大、老墨和馬克提供
// ============================================================

// ------------------------------
// 1. 變數宣告區
// ------------------------------

// ------------------------------
// 2. 邏輯判斷區
// ------------------------------

// ------------------------------
// 3. 腳本輸出區
// ------------------------------
```

## 二、命名規範

### 1. 變數（Variables）

- 採 **camelCase**，首字母小寫，其後單字首字母大寫
- 首字母一律以 `_` 開始，避免和保留字衝突
- 變數名稱要具描述性，避免縮寫（除非業界慣用）
- 期間、參數建議附註單位或用途
  - 正確：`_fastPeriod`, `_slowPeriod`, `_rsiThreshold`
  - 錯誤：`fp`, `sp`, `rt`
- 宣告結構規範 (重要)：
  - 統一區塊：所有自訂變數（包含數值、字串、以及布林值 `true`/`false`），一律必須在 `variable:` 或 `var:` 區塊中宣告。
  - 布林變數：用於條件判斷的布林變數（如 `_isGoldenCross`），請在 `var:` 中宣告並給予初始值 `true` 或 `false`。

### 2. 常數/Input（Constants）

- 採 **SCREAMING_SNAKE_CASE**，用於固定參數或不可變值
- 首字母一律以 `_` 開始，避免和保留字衝突
  - 例如：`_DEFAULT_PERIOD = 14`, `_MAX_BARS = 1000`

### 3. 函式（Functions/Subroutines）

- 採 **PascalCase**，首字母大寫
- 動詞開頭，表明行為
  - 例如：`CalculateMA()`, `GenerateSignal()`

### 4. 指標與策略名稱

- 同檔案名稱，PascalCase
- 與檔案一一對應

### 5. 嚴禁使用的名稱

- 僅為增加可讀性而被系統自動忽略的字（如 `a`, `is`, `on`, `the` 等），不可宣告為變數名稱。
- 系統內建常數（如 `pi`, `monday` 等），不可覆蓋或宣告為變數。

### 6. 陣列宣告的 1002 錯誤防呆

XS 系統對單一陣列的元素總數量有嚴格上限（約 7000 個元素以內）。計算公式為 `(第一維度 + 1) * (第二維度 + 1)`。例如 `array: _ticks[1000, 10](0);` 總數會達 11011 個，編譯時將觸發「1002 腳本物件產生失敗」。

**抓取 tick 的二維陣列，最安全的黃金比例寫法為 `array: _ticks[500, 10](0);`**

### 7. 逐筆洗價狀態保存 (`intrabarpersist`)

在逐筆洗價模式下，凡涉及「狀態紀錄」或在同一根 K 棒內的 tick 之間「累計數值」的變數，必須使用 `intrabarpersist` 關鍵字宣告。若漏寫，變數會在每次收到新 tick 時被重置回該根 K 棒開盤時的初始值。

### 8. rank 獨立空間宣告限制

`rank` 區塊是獨立空間，無法傳入腳本他處的參數和變數，也無法使用外部的 `input` 值。在 `rank` 內部需要建立並宣告專屬變數，不能與其他物件名稱衝突。

## 三、縮排與排版

- **使用 4 個空格**作為一層縮排，不使用 Tab（在vscode中可以設定tab=4個空格）
- 花括號與關鍵字同行，且保留一個空格

```xs
if condition then begin
    // 縮排 4 個空格
    setposition(1);
end;
```

- 運算子前後須保留一個空格

```xs
_value = close - open;
```

- 逗號後須空一格

```xs
_ma1 = average(close, _fastperiod);
```

## 四、註解（Comments）

- 單行註解使用雙斜線 `//`，置於程式碼行上方或行尾
- 多行說明使用區塊註解 `{ ... }`，適用於流程描述、多行說明或暫時註解程式碼
- 註解要完整說明「為何」而非「如何」

```xs
// 計算快線與慢線均線
_fastMA = average(close, _fastPeriod);
_slowMA = average(close, _slowPeriod);

{
  若快線向上突破慢線，則進場買入（強勢訊號）
}

if _fastMA crosses above _slowMA then begin
    setposition(1);
end;
```

## 五、行長度與換行

- 建議 **行長度不超過 100 字元**
- 引數過多時，可分行對齊，並在續行前加上 4 個空格縮排

```xs
plot1(
    _fastma,
    "快線",
    cyan
);
```

## 六、結構與流程控制

- 關鍵字（`if`、`for`、`while`、`begin`、`end`）皆小寫
- 條件式簡潔，複雜邏輯可拆為輔助變數

```xs
_isbullishcross = _fastma crosses above _slowma;
if _isbullishcross then begin
    setposition(1);
end;
```

- 避免過深巢狀，建議巢狀深度不超過三層

## 七、錯誤處理與邊界檢查

- 在使用歷史資料前，檢查資料長度

```xs
if currentbar < _slowperiod then
    return;  // 資料不足，退出
```

- 對於除零等例外情況，加入預設值或警告

```xs
if volume = 0 then
    _avgvolume = 0
else
    _avgvolume = totalvolume / volume;
```

### 資料取值防呆 (`default`)

務必在 `getfield` 後方加上 `default` 參數，避免資料缺漏時造成腳本中斷：

```xs
value1 = getfield("本益比", "d", default:=0);
```

### getsymbolfield 第一參數限制

`getsymbolfield` 的第一個參數**只接受字串常數、input 參數或 group 元素**，不接受 `var` 字串變數。必須直接傳入 group 元素：

```xs
// 錯誤：存入 var 再傳（編譯報錯：第 1 個參數應該是 String/Input/Group）
_cbid = _cbgroup[1];
_cbclose = getsymbolfield(_cbid, "收盤價");

// 正確：直接傳 group 元素
_cbclose = getsymbolfield(_cbgroup[1], "收盤價");
```

### 跨頻率取值正確寫法

若要取跨頻率的前期資料，必須直接對函數加上期數：

```xs
// 正確：直接對 getfield 加上中括號取前期值
value1 = getfield("月營收", "m")[1];

// 錯誤：先存入變數後再加中括號（會產生對位問題）
value1 = getfield("月營收", "m");
value2 = value1[1]; // 嚴禁此寫法
```

### 交易進場互斥判斷

進場時，務必同時判斷 `position = 0 and filled = 0`，代表「策略預期沒有部位，且實際也沒有部位」時才執行進場，以防重複送單。

### 停損停利價格基準

不要使用當時的 `close`，應使用 `filledavgprice`（實際成交均價）作為計算基準，避免滑價造成出場點計算錯誤。

### 運行環境防呆

為防止策略被放錯頻率執行，腳本第一行應加上頻率保護：

```xs
if barfreq <> "min" then raiseruntimeerror("僅支援分鐘頻率");
```

當沖腳本必須加入嚴格的時間範圍限制與尾盤平倉防呆機制。

### readticks 函數的絕對格式

盤中抓取逐筆明細的核心函數 `readticks` 具備嚴格限制：

- **參數限制**：僅接受 2 個參數：`readticks(陣列名稱, 紀錄序號的變數)`。禁止輸入 `date` 或 `time` 等舊版參數。
- **維度限制**：傳入陣列的第二維度**強制 >= 10**（塞入11個隱藏欄位），否則編譯失敗。
- **欄位對位**：`[i, 3]` 成交價、`[i, 4]` 單筆量、`[i, 5]` 內外盤標記 (1買/-1賣)、`[i, 10]` 連續成交總量。

### 換日歸零防呆機制

在逐筆洗價 (tick) 環境下，**絕對禁止依賴 `date[1]`** 做換日判斷。必須宣告獨立的 `intrabarpersist` 變數記錄「最後歸零日」：

```xs
var: intrabarpersist _reset_date(0);

if _reset_date <> date then begin
    // 執行各項累計數值的歸零動作...
    _reset_date = date; // 歸零後立即上鎖
end;
```

### 開盤集合競價（第一筆）處理

台灣股市 09:00:00 開盤第一筆為「集合競價」，內外盤標記通常為 0。`readticks` 回傳陣列索引 `[1]` 是「時間最新」的一筆，欲精準抓取時間序列上的「第一筆」，迴圈必須使用 `downto 1`（從最舊往最新讀取）：

```xs
var: intrabarpersist _has_first_tick(0);

if _reset_date <> date then begin
    _has_first_tick = 0;
    _reset_date = date;
end;

for _idx = _row_count downto 1 begin
    if _tick_array[_idx, 2] >= 090000 and _tick_array[_idx, 11] > 0 then begin
        if _has_first_tick = 0 then begin
            _bs_flag = 1; // 強制當作外盤
            _has_first_tick = 1;
        end else begin
            _bs_flag = _tick_array[_idx, 5]; // 後續看真實標記
        end;
    end;
end;
```

### 除錯輸出 (`print`)

開發期間建議使用 `print()` 將關鍵變數印出抓蟲。若在交易腳本中使用 `print`，必須透過 `file` 參數指定輸出檔案路徑與名稱。

## 八、資料取得函數替代規則

- 當需要取得日線級別的開高低收資料時，**禁止**使用以下函數：
  - `openD`、`closeD`、`highD`、`lowD`

- 應改用 `GetField` 函數搭配頻率參數 `"D"` 取代：

| 原函數 | 替代寫法 |
|--------|----------|
| `openD` | `GetField("Open", "D")` |
| `closeD` | `GetField("Close", "D")` |
| `highD` | `GetField("High", "D")` |
| `lowD` | `GetField("Low", "D")` |

```xs
// 錯誤寫法
_todayOpen = openD;
_todayHigh = highD;

// 正確寫法
_todayOpen = GetField("Open", "D");
_todayHigh = GetField("High", "D");
```

## 九、示範範例

### 選股腳本（四大區塊標準寫法）

```xs
// ============================================================
// 腳本名稱：外資連買且突破均線選股
// 邏輯說明：篩選外資連續買超3天，且今日收盤價突破自訂均線的股票
// ============================================================
// 此代碼參考資料感謝 XQ 達人園長、院長、貓老大、老墨和馬克提供
// ============================================================

// ------------------------------
// 1. 變數宣告區
// ------------------------------
input:
    _length(10, "均線天數");   // 外部參數：均線計算天期

var:
    _mavalue(0),               // 儲存均線數值
    _foreignbuy(0);            // 儲存外資買賣超張數

// ------------------------------
// 2. 邏輯判斷區
// ------------------------------
_mavalue = average(close, _length);
_foreignbuy = getfield("外資買賣超", "d");

condition1 = close cross over _mavalue;
condition2 = trueall(_foreignbuy > 0, 3);

// ------------------------------
// 3. 腳本輸出區
// ------------------------------
if condition1 and condition2 then begin
    ret = 1;
    outputfield1(_mavalue, "均線數值");
    outputfield2(_foreignbuy, "今日外資買超張數");
end;
```

### 交易腳本（四大區塊標準寫法）

```xs
// ============================================================
// 腳本名稱：均線交叉交易策略
// 邏輯說明：依據快慢均線交叉進行交易
// ============================================================
// 此代碼參考資料感謝 XQ 達人園長、院長、貓老大、老墨和馬克提供
// ============================================================

// ------------------------------
// 1. 變數宣告區
// ------------------------------
input:
    _fastperiod(20, "快線天數"),
    _slowperiod(50, "慢線天數");

var:
    _fastma(0),                // 快線均線
    _slowma(0),                // 慢線均線
    _isbullishcross(false);    // 是否黃金交叉

// ------------------------------
// 2. 邏輯判斷區
// ------------------------------
if currentbar < _slowperiod then return;

_fastma = average(close, _fastperiod);
_slowma = average(close, _slowperiod);
_isbullishcross = _fastma crosses above _slowma;

// ------------------------------
// 3. 腳本輸出區
// ------------------------------
if _isbullishcross and position = 0 and filled = 0 then begin
    setposition(1, label:="均線黃金交叉");
end;

plot1(_fastma, "快線");
plot2(_slowma, "慢線");
```

## 十、交付後使用指引

當完成 XS 程式碼撰寫並通過審查後，必須引導用戶完成以下步驟：

### 1. 匯入 XQ

提供兩種匯入方式讓用戶選擇：

- **方式 A（直接貼上）**：複製程式碼 → 打開 XQ 全球贏家 → 策略(D) → XScript 編輯器(E) → 新增腳本（類型須對應腳本類別）→ 貼上程式碼。
- **方式 B（開啟 .xs 檔）**：打開 XQ 全球贏家 → 策略(D) → XScript 編輯器(E) → 檔案 → 開啟 XS 檔案 → 選擇對應的 .xs 檔。

### 2. 編譯

告知用戶按 **F6** 或點選工具列「編譯」按鈕。編譯失敗時，請用戶將錯誤訊息貼回來協助修正。

### 3. 開始使用

根據腳本類別，告知用戶對應的啟用操作：

- **指標腳本**：點選「加入指標」，在技術分析圖上顯示
- **選股腳本**：點選「加入選股」，在選股中心使用
- **警示腳本**：點選「加入雷達」，在策略雷達中監控
- **交易腳本**：點選「加入自動交易」，設定自動下單策略

### 4. 詢問存檔

交付後必須詢問用戶：「是否要另存成 .xs 檔案？」

- 用戶選「是」→ 詢問存檔路徑，將程式碼存為 .xs 檔，並提示可用方式 B 匯入
- 用戶選「不用」→ 結束
